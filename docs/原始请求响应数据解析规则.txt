  req-resp.html 完全在前端 JS 中实时解析，数据来源是后端 /api/req-resp/records
   返回的原始 proxy JSONL 记录（非 clear-req-resp 输出）。

  1. 左侧列表展示

  从每条 raw record 提取：
  - _message_id：后端服务器注入，从 sse_events 的 message_start 事件里取
  message.id
  - timestamp、method、url、response.status、is_sse、sse_events.length

  2. 请求 Body（右侧 4 个 tab）

  取 r.request.body，尝试 JSON.parse：
  - 消息：request_json.messages 最后一条，Markdown 渲染
  - System：request_json.system，Markdown 渲染
  - Tools：request_json.tools[]，展示 name/description/input_schema
  - 原始：完整 JSON tree（默认展开 3 级）

  3. SSE 响应解析（parseSseEvents）

  遍历 sse_events[]，按事件类型累积：

  ┌─────────────────────────────────────┬─────────────────────────────────┐
  │              事件类型               │              处理               │
  ├─────────────────────────────────────┼─────────────────────────────────┤
  │ content_block_start                 │ 记录 block                      │
  │                                     │ 元信息（type/name/id）          │
  ├─────────────────────────────────────┼─────────────────────────────────┤
  │ content_block_delta / text_delta    │ 累积 text                       │
  ├─────────────────────────────────────┼─────────────────────────────────┤
  │ content_block_delta /               │ 累积 partial_json（tool input） │
  │ input_json_delta                    │                                 │
  ├─────────────────────────────────────┼─────────────────────────────────┤
  │ content_block_stop                  │ 标记 block 完成                 │
  ├─────────────────────────────────────┼─────────────────────────────────┤
  │ message_delta                       │ 取 stop_reason、usage           │
  └─────────────────────────────────────┴─────────────────────────────────┘

  输出：{ contents: [{type, name, id, text, input}], stopReason, usage }

  两个 tab 展示：
  - 解析：按 block 展示 text（Markdown）或 tool_use（JSON tree）
  - 原始：每条 event 原文 + 行内注释（buildSseEventAnnotations）
