# 前端界面优化清单

## Detail 面板重构（P0 - 最高优先级）

基于参考界面，重新设计 Detail 面板布局：

### 整体结构（从上到下）

```
┌─────────────────────────────────────┐
│ 标题栏 (Turn 类型 + 时间 + ID)        │
├─────────────────────────────────────┤
│ 日志元数据 (可折叠)                  │
│   - uuid, parentUuid, promptId      │
│   - sessionId, permissionMode       │
│   - cwd, version                    │
│   - Token 统计 (保持标签格式)        │
├─────────────────────────────────────┤
│ 动态内容区 (根据 Turn 类型变化)       │
│   - user_message: 消息内容          │
│   - assistant_text: Thinking + Text │
│   - tool_use: Tool Call 详情        │
│   - tool_result: Tool Result 详情   │
├─────────────────────────────────────┤
│ 原始 JSON (可折叠)                   │
│   - 完整 entry 原始数据              │
└─────────────────────────────────────┘
```

### 具体设计规范

- [x] **1. 日志元数据区域**
  - 可折叠设计，默认展开
  - 字段：uuid、parentUuid、promptId、sessionId、permissionMode、cwd、version
  - Token 统计保持现有标签格式（不采用横向平铺）
  - 样式：键值对两列布局

- [x] **2. 动态内容区 - user_message**
  - 标题："消息内容"
  - 展示：用户发送的完整消息文本
  - 支持 Markdown 渲染

- [x] **3. 动态内容区 - assistant_text**
  - Thinking 区域（如有）：独立折叠面板，标题 "Thinking"
  - Text 区域：标题 "回复内容"
  - 两者都支持 Markdown 渲染

- [x] **4. 动态内容区 - tool_use**
  - 标题："Tool Call"
  - 展示：工具名称、input 参数（JSON 格式化）
  - 支持语法高亮

- [x] **5. 动态内容区 - tool_result**
  - 标题："Tool Result"
  - 展示：结果内容（根据类型渲染，JSON/文本/Markdown）
  - 支持语法高亮

- [x] **6. 原始 JSON 区域**
  - 可折叠设计，默认折叠
  - 展示：当前 entry 的完整原始 JSON
  - 支持语法高亮

- [x] **7. Markdown 渲染支持**
  - 代码块、列表、粗体、链接
  - 代码块支持语法高亮和复制按钮

## 其他优化项

### 显示内容优化

- [x] **8. Turn/Entry 统计信息歧义**
  - 问题：统计显示 "Turn 20 / Entry 22"，左侧树显示 "12 steps"
  - 方案：改为显示 "Steps 12 / Total Turns 20 / Entries 22"

- [x] **9. Diff 功能不展示 thinking 内容**
  - 问题：Turn Diff 弹窗中 THINKING 标签显示 "(无)"
  - 方案：修复 Diff 功能，正确提取并展示 thinking 内容

### 排版与布局优化

- [x] **10. 左侧树支持批量折叠/展开**

- [x] **11. 代码块样式优化**
